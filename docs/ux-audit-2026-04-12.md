# AIPM 使用侧审计 — 2026-04-12

> **审计人**: Claude (模拟 ADMIN 用户，走过 10 个主要模块)
> **环境**: feat/learning-copilot 分支，含 demo seed 数据
> **方法**: 真实点击 + 源码对照，不是静态 review

---

## 0. 核心结论

**这是两个产品粘在一起，不是一个产品。**

### 两个产品并存

| | 旧产品（SOP 驱动执行） | 新产品（假设驱动学习，你要的） |
|---|---|---|
| 哲学 | **建 IP**：SOP / 提示词 / Skill 沉淀复用 | **跑实验**：假设 → 验证 → 学习 |
| 入口 | 知识中心、任务管理、工作台、团队看板 | 目标洞察、目标管理、假设模板库、学习库 |
| 核心动词 | 套用模板 → 生成工作包 → 执行 | 提假设 → 关闭 → 读 learning |
| 衡量 | 贡献点（金银铜牌） | 学习速率 / 胜率 / KR 达成度 |
| 模板 | SOP 模板（文档包） | 假设模板（Statement 占位符） |
| 人员 | 需求架构师 + 实施工程师 | Hypothesis Owner + KR Owner |

**用户看见的症状**：
- 侧栏 10 项菜单交错排列（新/新/新/旧/旧/旧/旧/新/新/设置）
- 术语分裂（"模板" / "任务" / "看板" 各有两套含义）
- 没有 bridge：看见问题的地方没有 CTA，想做动作的地方没有数据

这是割裂感的根源。不解决这个，单点修修补补意义不大。

## 2. 流程断点 — 按旅程逐条记录

### F1：我打开首页，我能干什么？

**路径**: 登录 → `/dashboard`

| # | 观察 | 严重度 |
|---|---|---|
| F1.1 | Dashboard 只有 2 个 button（重新生成 Copilot + 用户头像），**没有任何主 CTA** | 🔴 严重 |
| F1.2 | **KR 卡片不可点击** — 看见"首周付费转化 严重落后"却没法进入该 KR 或查看其假设 | 🔴 严重 |
| F1.3 | **Copilot 建议卡片不可点击** — AI 说"建议追加 3 个视频变体"但没按钮"为我创建这 3 个假设" | 🔴 严重 |
| F1.4 | **Alert 里的 evidenceHypothesisIds 是死代码** — 后端返了，前端 render 时没 click handler 跳转到证据假设 | 🟡 重要 |
| F1.5 | 本周 hypothesis 4 bucket 里的 item **可点击**跳详情，但 bucket header 里的"胜出"、"失败"计数不可点击（不能 filter 查看） | 🟢 小 |
| F1.6 | 学习速率 "1"、胜率 "100%" 是 static number，点不了也没有趋势图 | 🟢 小 |

**代价**: 用户在 Dashboard 识别到 3 个 KR 有问题、5 条 Copilot 建议，但**每一个动作都需要手动切 sidebar → /hypotheses → 新建 → 复制粘贴 statement → 选 KR**。洞察到动作的距离是 4-6 步点击。

### F4：我想看某个假设详情 + 加 A/B 变体

**路径**: /hypotheses → 点某行 → /hypotheses/:id

| # | 观察 | 严重度 |
|---|---|---|
| F4.1 | 详情页 header 没有"**主要动作 CTA**" — 假设是 CLOSED_WIN 时就没事可做吗？应该有"加后续迭代"、"另存为模板"、"导出分享" | 🟡 重要 |
| F4.2 | 4 个 tab（概览 / A/B 变体 / 迭代历史 / 学习库）**命名不一致** — "学习库" tab 和 sidebar 的"学习库"同名但指的是**本假设的** learnings，不是全局库。令人混淆 | 🟡 重要 |
| F4.3 | A/B variants tab **空状态没引导** — "还没有 A/B 变体" + 一个按钮，没告诉用户 A/B 测试是干嘛的、什么时候需要 | 🟢 小 |
| F4.4 | 加 variant 时只有 name + description，**没有 hypothesis 文案区分** — 两个 variant 都绑定到同一 statement，真正的 A/B 是比较**不同方案**，应该在 variant 里记录"方案说明" | 🟡 重要 |
| F4.5 | 录入数据 dialog 只有 sampleSize + conversionCount，**不支持其他 metric type**（duration、revenue 等），但 schema 里有这些枚举值 | 🟡 重要 |
| F4.6 | Mark winner 只对 isSignificant=true 的 variant 可见，**没有 force 覆盖选项** — 违反 PRD US-6.3（显著性不够时可 force + audit log） | 🟡 重要 |
| F4.7 | ICE 分数显示 "ICE: 33.6 (impact=8 × conf=7 × ease=6)/10" — 这是开发自语，用户看不懂"33.6 是好还是坏" | 🟡 重要 |

**代价**: 详情页是最重的功能区，但把 ICE 展示成内部公式 + tab 命名冲突 + A/B variant 语义错位 = 用户不敢深入。

## 3. 按优先级打包的修复建议

### 🔴 P0 — 阻塞核心流程的断点

1. **Copilot 建议一键采纳** — nextHypothesisSuggestion 卡片加"采纳此建议"按钮 → 用预填的 statement/mechanism/krId 打开新建假设 dialog → 用户只需确认。**这是 Learning Copilot 最核心的 KPI 入口**，不做等于选项 C 没实现。
2. **KR 详情页 + KR CRUD** — 创建 KR 必须有 UI，KR 详情页聚合该 KR 的所有假设（RUNNING / CLOSED_WIN / CLOSED_LOSS 三列 kanban）+ 历史趋势图。Dashboard 的 KR 卡片点击进入此页。
3. **新建假设支持"从模板开始"** — PRD US-5.1 明确要求。新建 dialog 顶部加 tab："从模板 / 空白"，默认"从模板"。
4. **Alert/Pattern 证据可点击** — CopilotAlert 里的 evidenceHypothesisIds、Pattern 里的 evidenceHypothesisIds 点击跳假设详情。
5. **IA 重组** — 菜单按产品哲学分组：
   ```
   洞察与决策（新）
     ├─ 目标洞察
     ├─ 跨项目对比 (ADMIN)
   目标与实验（新）
     ├─ 目标管理
     ├─ 假设模板库
     ├─ 学习库
   执行与协作（旧/要么合并要么 archive）
     ├─ 任务管理
     ├─ 工作台
     ├─ 团队看板
   资源库（旧）
     ├─ 知识中心
   设置
   ```

### 🟡 P1 — 重要但不阻塞

6. **Hypothesis ↔ Iteration 双向链接** — 详情页显示"本假设在哪些 iteration 下执行"；任务管理的 iteration 卡显示关联的 hypothesis
7. **详情页 4 tab 改名** — "学习库" tab → "本假设学习笔记"（消除和 sidebar 同名）；或整个 tab 内联到概览
8. **ICE/RICE 打分从数字变决策辅助** — 不显示 `(impact=8 × conf=7 × ease=6)/10`，显示 "高 / 中 / 低 / 不推荐"，并和当前项目其它假设比较 percentile
9. **新建假设 dialog 支持 ICE/RICE** — 创建时就能打分，不用进详情再加
10. **学习库按 KR/tag/时间筛** + learning card 可跳回假设
11. **关闭假设 dialog prefill expectedImpact** — 不让用户重填
12. **A/B variant 语义澄清** — variant 不只是 name，应该是"方案说明 + 链接到实际设计/代码 commit/feature flag"
13. **模板 usageCount / 胜率回写**
14. **另存为模板**（US-5.2）

### 🟢 P2 — 打磨项

15. KR 进度用 time-based progress（按 startDate→endDate 画时间轴）
16. Learning 日期加相对时间
17. 关闭后轮询状态有视觉指示（spinner + "AI 正在学习"）
18. Template placeholder key 和 label 对应关系在 UI 上可见
19. 一键关闭 dialog 中 metricType 支持全部枚举
20. Dashboard 学习速率/胜率加小 trend chart

### 🧱 结构级讨论（P0 之前要决定）

这个必须选 ONE：

- **A. 硬切新产品**：把旧的 4 个模块（团队看板/任务管理/工作台/知识中心）archive 掉，这个产品只做 Learning Copilot。**最干净**，但放弃了 SOP/提示词/Skill 的价值。
- **B. 整合两个产品**：把 Hypothesis 作为 Iteration 的 parent（或反之），SOP 模板套用时自动绑定到一个 hypothesis，Feed 包执行完成后反馈到 hypothesis.result。团队看板的贡献点加入"胜出的 hypothesis / 高质量 learning"类别。**最贵**，但真正统一。
- **C. 标签化共存**：在侧栏用"Learning Copilot" 和 "SOP 工作流" 两个分组，让用户自己选工作模式，互不打扰。**最便宜**，但承认割裂。

**我的推荐：B（整合）**，但分两步：
1. P0 先做新产品内部的 5 条（上面列的），让新产品能闭环
2. P1 再做 Hypothesis ↔ Iteration 桥接（只做这一条就足够 80% 整合效果）
3. 团队看板的激励规则改造放到后续

## 4. 可量化验证

修复完成后，用户在 dashboard 看见一个 "僵死 KR + 连续失败方向" → **3 次点击内**应能：
1. 点 Copilot 建议 → "采纳" → 打开预填 dialog
2. 改一行确认 → 创建假设
3. （关闭时）→ 看到 AI learning

当前状态：**9+ 次点击**（sidebar 切模块 2 次 + 新建 → 手填 7 个字段 + 确认 + 关闭流程）。

减少 60%+ 点击是可量化的北极星。
